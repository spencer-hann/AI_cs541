We can see that the agent is acting rationally with respect to its percept sequence. Whenever it perceives dirt it performs the Suck action, regardless of its location. As a performance measure, I decided to use the number of clean squares after the agent had had 100 turns. To build the simple reflex agent, a used a hash table to manually map certain locations to the locations the agent would move to next if the square was clean. It would simply check to see if a square was dirty, then either clean the square or perform a lookup to get its next location. The movement table just sent the agent around the edges of the grid, with the middle cell just mapped to an edge cell. This way, the agent could continue in a cycle, reaching 8 of the 9 squares, 9 of the 9 if it happened to initialize in the center square.

​ To build the random agent, I just used a random number generator to decide actions. Finally, a number in the range of was added to the agent's row or column value, which would then be verified as still within the matrix. A in this final stage represented the action.

Without any significant level of "intelligence", but rather, just the ability to perceive some basic information in the environment and react to it, my agent was able to perform well at this simple task. One easy way to improve this agent would be to allow it to store information internally, beyond its own location and movement table.
